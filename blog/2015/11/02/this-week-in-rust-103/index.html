This Week in Rust
issue 103 — 02 NOV 2015

Hello and welcome to another issue of This Week in Rust! Rust is a systems language pursuing the trifecta: safety, concurrency, and speed. This is a weekly summary of its progress and community. Want something mentioned? Tweet us at @ThisWeekInRust or send us an email! Want to get involved? We love contributions.

This Week in Rust is openly developed on GitHub. If you find any errors in this week's issue, please submit a PR.

Notable changes

  • Port the standard crates to PNaCl/NaCl.
  • Fix excessive memory allocation in RawVec::reserve.
  • Implement conversion traits for primitive float types.
  • Generalise associative operator parsing.
  • Do not allow a module and tuple struct of the same name to coexist.
  • Allow constant evaluation of function calls.
  • Remove dependency on libgcc-dw2-1.dll from win32 executables.
  • Remove contraction from region inference.

Approved RFCs

Changes to Rust follow the Rust RFC (request for comments) process. These are the RFCs that were approved for implementation this week:

  • RFC 1291: Promote the libc crate from the nursery.
  • RFC 1307: Add some additional utility methods to OsString and OsStr.

Final Comment Period

Every week the team announces the 'final comment period' for RFCs and key PRs which are reaching a decision. Express your opinions now. This week's FCPs are:

  • Allow overlapping implementations for marker traits.
  • Enable the compiler to cache incremental workproducts.
  • #[deprecated] for Everyone.
  • Improvements to the Time APIs.
