This Week in Rust 122

Hello and welcome to another issue of This Week in Rust! Rust is a systems language pursuing the trifecta: safety, concurrency, and speed. This is a weekly summary of its progress and community. Want something mentioned? Tweet us at @ThisWeekInRust or send us an email! Want to get involved? We love contributions.

Notable New Crates & Project Updates

  • Library stabilizations for the 1.9 release.
  • Multirust 0.8 now enables installation of additional standard libraries for cross compilation.
  • regex crate can match regular expressions on arbitrary bytes.
  • Pencil. A web application microframework for Rust.
  • preferences. Read and write user-specific application data in Rust.
  • Proteus. Axolotl crypto protocol implementation in Rust.
  • libterm. A pure Rust library for handling, manipulating and reading information about terminals. This provides a full-featured alternative to Termbox.
  • dryad. An almost-parallel, semi-functioning, dynamic linker experiment, written in Rust.

Final Comment Period

Every week the team announces the 'final comment period' for RFCs and key PRs which are reaching a decision. Express your opinions now. This week's FCPs are:

  • Add support for naked functions.
  • Expand the current pub/non-pub categorization of items with the ability to say "make this item visible solely to a (named) module tree".
  • Add octet-oriented interface to std::net::Ipv6Addr.
  • Unix socket support in the standard library.
  • Implement a method, contains(), for Range, RangeFrom, and RangeTo, checking if a number is in the range..
  • Add a replace_slice method to Vec<T> and String which removes a range of elements, and replaces it in place with a given sequence of values.

New RFCs

  • Saturating and checking integer wrapper types.
  • Stabilize the -C overflow-checks command line argument.
  • Revise type ascription operator to use type equality, not coercion.
  • Add TryFrom and TryInto traits.
  • Add more integer atomic types.
