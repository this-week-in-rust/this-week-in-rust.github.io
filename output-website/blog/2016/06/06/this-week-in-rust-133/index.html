This Week in Rust 133

Hello and welcome to another issue of This Week in Rust! Rust is a systems language pursuing the trifecta: safety, concurrency, and speed. This is a weekly summary of its progress and community. Final Comment Period

Every week the team announces the 'final comment period' for RFCs and key PRs which are reaching a decision. Express your opinions now. This week's FCPs are:

  • Implement new methods for checked and wrapping casts for potentially lossy integer conversions.
  • Change thread local variables to only accept async-signal-safe types.
  • Add a initial, minimal form of impl Trait.
  • Normalization for long error codes explanations.
  • Standardise stream wrappers like compression, encryption.
  • Add a lifetime specifier to macro_rules!.
  • Remove the one-type-only restriction on format_args! arguments.

New RFCs

  • Require documentation for all new features.
  • Automatically implement some traits for !.
  • Add two new pointer-sized integer types; uptr and iptr.
  • Add checked_sub() already known from various primitive types to the Duration struct.
  • Add PTX and AMDGPU targets.
